## 三、资料查阅与实践过程

| 时间 | 学习主题 | 资料来源 | 学习收获 |
| --- | --- | --- | --- |
| 2026-07-03 | Bootstrap 重采样原理复习 | 教材 Efron & Tibshirani (1993)《An Introduction to the Bootstrap》Ch.6–8 + 网络资料 | 复习了 Bootstrap 的核心思想：有放回抽 $n$ 个、重复 $B$ 次、用统计量经验分布的分位数做置信区间；明确它不假设分布族，适合没有闭式区间的统计量。 |
| 2026-07-03 | 估计目标与数据集选型 | Proschan (1963) *Technometrics* 5(3):375–383；R `boot` 包 `aircondit7` 数据集文档 | 选定估计**总体中位数 $m$**；数据用 `aircondit7`（波音 720 第 7 架机空调故障间隔，n=24，右偏小样本）；另用 Exp(1) 模拟（真中位数 $\ln 2$ 已知）做覆盖率对照。 |
| 2026-07-03 | 为何选中位数（方法选择核心） | 课程任务书 + 中位数渐近 SE 公式推导 | 中位数 $\mathrm{SE}\approx 1/(2f(m)\sqrt n)$ 依赖未知密度 $f(m)$，没有均值（$t$ 区间）、比例（Wilson）那样的现成公式——理清了 Bootstrap 相对其它区间方法的适用边界，理解中位数场景为何非它不可。 |
| 2026-07-03 | 方案讨论：三层结构与主辅方法取舍 | 开题选题会讨论 + Efron 1993 Ch.13–14 | 敲定「数学模型层 → 计算实现层 → 可视化验证层」三层结构；方法上取 percentile 简单稳妥做主、BCa 做对照（不取 BCa 主，因 percentile 已够用且更直观）；覆盖率实验用 Exp(1) 已知真值自证方法可信。 |
| 2026-07-04 | percentile 与 BCa 公式核验 | Efron 1993 eq 13.5、eq 14.10 / 14.15；`scipy.stats.bootstrap` 文档 | 核对 percentile 取 2.5% / 97.5% 经验分位数；BCa = 偏差修正 $\hat z_0$ + jackknife 加速度 $\hat a$；确认 $\hat z_0$ 用严格「$<$」计数（与 scipy 一致）。 |
| 2026-07-04 | 自主学习：numpy 向量化与 scipy API | numpy / scipy 官方文档 + 示例代码 | 学会 `rng.integers` 生成索引矩阵、`np.median(axis=1)` 批量算统计量、`np.partition` 做 $O(n)$ 选择；并对照 `scipy.stats.bootstrap` 的 BCa 实现核验自写代码。 |
| 2026-07-04 | 蒙特卡洛误差与覆盖率实验设计 | Efron 1993 Ch.19 + 网络资料 | 学习覆盖率实验的 MC 标准误：事前按名义 $p\approx0.95$ 估 $\sqrt{0.95\times0.05/1000}\approx0.7$ 个百分点、据此定 $R=1000$；事后报告一律按观测 $\hat p$ 重算（实测 $\approx0.79$ 个百分点），不套 $p=0.5$ 最坏上界。 |

## 四、问题记录与解决

| 时间 | 问题描述 | 解决过程 | 结果与反思 |
| --- | --- | --- | --- |
| 2026-07-04 | $B=10000$ 次重采样用 Python `for` 循环逐次抽样 + 算中位数，单次分析要跑几十秒，覆盖率实验（$R\times B$）更跑不动 | 改用 numpy 向量化：`rng.integers` 一次生成 $(B,n)$ 整数索引矩阵，`np.median(axis=1)` 一次性算出 $B$ 个 bootstrap 中位数；中位数内部用 `np.partition` 保持 $O(n)$ | $B=10000$ 秒级跑完。教训：numpy 下百万级运算必须向量化，不能写 Python 层循环。 |
| 2026-07-05 | BCa 区间和 percentile **几乎重合**（BCa [22.0, 75.5] 与 percentile [22.0, 79.0]），第一反应是代码写错了 | 推导后发现不是 bug：偶数 $n$ 下留一法中位数只取 2 个值 $\{x_{(n/2)},x_{(n/2+1)}\}$，导致加速度分子 $\sum(\cdot)^3 = 0$（偶数 $n$ 无结时精确为 0）、$\hat a\equiv 0$；对 $n\in\{12,24,48,96\}$ 扫描验证都一样 | 确认是中位数（非光滑泛函）的固有性质，BCa 退化为「percentile + 偏差修正」。反思：先怀疑代码、再怀疑方法，本题恰是方法本身的性质，报告须如实承认局限。 |
| 2026-07-05 | Exp(1) 覆盖率 **0.934 < 名义 0.95**，一度怀疑方法不对 | 查文献确认 percentile 一阶精度 $O(n^{-1/2})$ 在偏态数据下本就轻度欠覆盖；MC 误差按观测 $\hat p$ 算约 0.79 个百分点；用解析标尺 SE≈$1/\sqrt{24}$≈0.204、区间宽≈0.80 与实测 0.796 交叉验证吻合 | 是发现不是 bug。教训：模拟结果要有解析标尺交叉验证，不能只盯一个数。 |
| 2026-07-06 | 在 `mission3/` 文件夹里跑 `python -m mission3.run_all` 报 `No module named 'mission3'`，数据相对路径也跟着错 | 这是 Python 包机制限制——模块错误本身无法在代码里消解，改为按运行位置切换命令：仓库根用 `python -m mission3.run_all`、`mission3/` 内用 `python -m run_all`；同时在 `run_all` 入口做 cwd 无关处理（自动定位仓库根与 `data/` 目录、数据加载改用基于包位置的安全路径），让数据路径不再依赖当前目录 | 从仓库根或 `mission3/` 内都能一键跑（分别用对应命令）。教训：可复现性不止固定随机种子，「在哪运行都能跑」也是基本工程要求。 |

## 七、总结与反思

通过本次任务三的实践，主要收获如下：

（1）**数学建模与统计思维能力**：把「估计一批右偏寿命数据的总体中位数」这个没有现成区间公式的问题，用 Bootstrap 解决。体会到中位数之所以最该用 Bootstrap，是因为它的标准误依赖未知密度 $f(m)$——这不是凑选题，而是从「哪些统计量有闭式区间、哪些没有」自然推出来的。

（2）**编程实现能力**：用 Python + numpy 从零实现了重采样、percentile、BCa、覆盖率实验四个模块。过程中遇到了 $B=10000$ 循环太慢的问题，通过向量化（`rng.integers` 生成索引矩阵 + `np.median(axis=1)`）解决；也遇到了在子目录运行报「No module named」的问题，通过 cwd 无关处理解决。

（3）**统计实验设计能力**：学会用 Exp(1) 已知真值做覆盖率对照来「自证」方法可信，并用解析标尺（SE≈$1/\sqrt{24}$）交叉验证模拟结果；理解了蒙特卡洛标准误要按观测 $\hat p$ 算，不能套 $p=0.5$ 的最坏上界。

（4）**科学可视化表达能力**：用 matplotlib 绘制了原始样本分布、Bootstrap 分布演化、CI 端点稳定曲线、CI 对比、三因素对比共五张图，理解了结构 / 过程 / 结果三类图各自的定位，每张图都配编号 + 中英标题 + 图注三件套，形成完整的可视化论证链条。

不足与改进方向：
- 中位数（非光滑、偶数 $n$）下 BCa 退化（$\hat a\equiv0$），没有真正展示 BCa 相对 percentile 的二阶优势；后续可加 10% 截尾均值（光滑统计量）做伴生演示。
- 真实数据 $n$ 固定为 24、无法扩样，样本量因素只能在 Exp(1) 模拟上做。
- Streamlit 仪表盘目前只本地运行，后续可部署到 Community Cloud，方便评委直接体验。
- 后续统稿阶段需注意与任务一、二的图表风格统一，确保报告整体一致性。
