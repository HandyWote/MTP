## 三、三个任务：原始要求与初步设计

### 任务一：迭代法解线性方程组（满分 15）

**原始要求（任务书）：**
- 将一个实际或模拟问题表示为 Ax=b，说明 A、x、b 的含义
- 至少实现一种迭代法（Jacobi / Gauss-Seidel）
- 设置初值、误差阈值、最大迭代次数，记录迭代过程
- 与直接求解结果对比，分析误差和收敛
- 结果说明：迭代是否收敛、收敛速度、误差是否达标、**矩阵性质对迭代的影响**

**初步设计框架：**
- **数学模型层**：把工程问题离散化为 Ax=b；明确 A/x/b 物理含义；分析矩阵性质（严格对角占优 → 保证 Jacobi / GS 收敛）
- **计算实现层**：同时实现 Jacobi 与 Gauss-Seidel；初值 x₀=0；停止条件 ‖xₖ−xₖ₋₁‖∞ < 1e-6 或达 max_iter；用 `numpy.linalg.solve` 求直接解作参考；记录每步误差与残差
- **可视化层**：
  - 结构：矩阵稀疏结构图 / 元素分布热力图
  - 过程：误差随迭代次数曲线（半对数）、Jacobi vs GS 收敛对比
  - 结果：数值解 vs 直接解对比、两方法迭代次数 / 误差 / 耗时对比
- **结果分析**：收敛性、收敛速度（与谱半径联系）、误差达标情况、矩阵性质如何决定迭代效果

⚠️ **待定：工程场景**（见第七节）

### 任务二：三控开关设计与实现（满分 15）

**原始要求（任务书）：**
- 三个开关 S1 / S2 / S3 ∈ {0,1}，枚举全部 8 种状态，建真值表
- 设计输出 L，写出布尔逻辑表达式
- 程序实现：输入开关状态 → 输出灯状态
- 验证：任一开关状态变化时，输出符合设计

**逻辑（已锁定）：** 由"任一开关变化 → 输出变化"得 `L = S1 ⊕ S2 ⊕ S3`（三开关异或，等效"按下奇数个开关则灯亮"）。

**初步设计框架：**
- **状态与逻辑建模**：8 种状态真值表；布尔表达式 L = S1⊕S2⊕S3；证明任一开关翻转 → L 翻转
- **计算实现**：枚举 8 状态自动计算 L；交互式输入输出；自动验证"翻转任一开关 L 必变"
- **可视化层**：
  - 结构：8 状态空间图（3 维立方体顶点）、真值表颜色矩阵、逻辑门 / 布尔表达式结构图
  - 过程：状态切换路径图（翻转一个开关的连边）
  - 结果：正确性验证表、输出状态分布
- **结果分析**：逻辑如何由状态空间与真值表导出、表达式是否简洁（异或最简）、程序输出与真值表一致性

✅ 本任务逻辑确定，无需开题会决策。

### 任务三：Bootstrap 方法解决估计问题（满分 15）

**原始要求（任务书）：**
- 选择一个估计问题，明确样本数据、统计量、估计目标
- 有放回重采样，重复多次生成 Bootstrap 样本
- 计算每次统计量，形成 Bootstrap 估计分布
- 计算点估计与置信区间
- 分析样本量、重采样次数、数据波动对估计的影响
- 结果说明：估计含义、置信区间意义、分布稳定性、**Bootstrap 适合该问题的原因**

**初步设计框架：**
- **问题与数据**：选定估计目标（⚠️待定，推荐中位数）；真实小数据集（n≈20–30）+ 模拟数据对照（验证 CI 是否覆盖真值）
- **Bootstrap 实现**：有放回重采样 B=10000 次，每次计算统计量，形成分布
- **区间估计**：点估计 + percentile 法 95% 置信区间
- **可视化层**：
  - 结构：原始样本分布图 / 箱线图 / 采样结构示意
  - 过程：统计量随重采样次数 B 的变化曲线（稳定性）、Bootstrap 分布演化
  - 结果：Bootstrap 分布直方图 + 密度、置信区间带图、原始估计 vs Bootstrap 估计对比
- **结果分析**：估计结果含义、CI 意义、分布稳定性、样本量 / B 的影响、为何 Bootstrap 适合该统计量

⚠️ **待定：估计目标与数据来源**（见第七节）
