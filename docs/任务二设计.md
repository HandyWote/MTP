# 任务二设计：三控开关的设计与实现

> 课程：数学思维实践（CST4822A）· 任务二 · 离散数学实验（满分 15）
> 版本：v0.1 提案　|　状态：**已锁定（逻辑确定）**
> 逻辑方案：**三异或 —— L = S1 ⊕ S2 ⊕ S3**

---

## 一、一句话目标

将"三个开关任意控制同一盏灯"的工程需求，抽象为离散状态空间和布尔逻辑，用程序实现并自动验证"拨动任一开关 → 灯状态翻转"的设计约束。

---

## 二、工程画面（这是什么问题）

```

         开关 S1           开关 S2           开关 S3
           │                │                │
           ├────────────────╋────────────────┤
           │         三控开关逻辑            │
           │        L = S1 xor S2 xor S3    │
           └────────────────╋────────────────┘
                            │
                         灯 L
                     ● = 亮 (L=1)
                     ○ = 灭 (L=0)
```

- **工程需求**：楼道里三个不同位置的开关，共同控制同一盏灯。任意拨动一个开关，灯的状态必须改变（亮变灭、灭变亮）。
- **数学本质**：寻找一个布尔函数 L = f(S1, S2, S3)，使得翻转任意一个输入变量时输出必然翻转。
- **核心洞察**：异或运算 XOR 满足"各位独立翻转则整体翻转"——这正好就是三控开关需要的性质。

## 三、状态与逻辑建模（评分：状态建模 3 分 + 逻辑设计 4 分 = 7 分）

### 3.1 变量定义

| 符号 | 类型 | 含义 |
|------|------|------|
| S1, S2, S3 | ∈ {0, 1} | 三个开关的状态（0 = 关，1 = 开） |
| L | ∈ {0, 1} | 灯的状态（0 = 灭，1 = 亮） |

### 3.2 状态空间

三个二值变量，全部可能的输入组合共 2³ = 8 种。将 8 种状态映射到三维立方体的顶点，(S1, S2, S3) 作为坐标。立方体的每条棱代表翻转一个开关——沿棱移动一步，恰好改变一个开关的取值。

### 3.3 真值表

由设计要求"任一开关变化 → 输出变化"，逐行推导 L 的值。约定初始状态全部关闭时灯灭（L(0,0,0) = 0），则：

| S1 | S2 | S3 | L | 说明 |
|----|----|----|---|------|
| 0  | 0  | 0  | 0 | 全部关 → 灯灭（初始约定） |
| 0  | 0  | 1  | 1 | 翻转 S3 → L 必须变 |
| 0  | 1  | 0  | 1 | 翻转 S2 → L 必须变 |
| 0  | 1  | 1  | 0 | 翻转 S1 → L 必须变 |
| 1  | 0  | 0  | 1 | 翻转 S1 → L 必须变 |
| 1  | 0  | 1  | 0 | 翻转 S2 → L 必须变 |
| 1  | 1  | 0  | 0 | 翻转 S3 → L 必须变 |
| 1  | 1  | 1  | 1 | 翻转 S3 → L 必须变 |

### 3.4 布尔表达式

观察真值表：L=1 的行恰好是三个开关之和为奇数的行。这对应**异或运算**：

$$L = S1 \oplus S2 \oplus S3$$

解释：奇数个开关被按下时灯亮，偶数个时灯灭。

若展开为标准与或式（SOP），则：

$$L = S1 \cdot \overline{S2} \cdot \overline{S3} + \overline{S1} \cdot S2 \cdot \overline{S3} + \overline{S1} \cdot \overline{S2} \cdot S3 + S1 \cdot S2 \cdot S3$$

显然异或形式更简洁。异或的可结合性保证了三控开关可拆解为级联结构：先用两个开关做一组异或，其结果再与第三个开关异或。

### 3.5 正确性证明

**命题**：对任意 (S1, S2, S3)，翻转任一 Si 后，L 的值必然取反。

**证明**：L = S1 ⊕ S2 ⊕ S3。翻转 S1 后，新值 L' = (¬S1) ⊕ S2 ⊕ S3 = ¬(S1 ⊕ S2 ⊕ S3) = ¬L。由异或运算的对称性，翻转 S2 或 S3 同理。证毕。

### 4.1 功能模块

| 模块 | 功能 | 核心代码 |

| 真值表生成 | 枚举 8 种状态，自动计算 L | `itertools.product([0,1], repeat=3)` + `s1 ^ s2 ^ s3` |
| 交互输入 | 用户输入三个 0/1 → 输出灯状态 | 带输入校验，`q` 退出 |
| 自动验证 | 8 状态 × 3 开关 = 24 种翻转，逐一验证 L 是否翻转 | 翻转前后 L 值对比，全部通过打印 PASS |
| 可视化 | 生成 4 张图表并保存 | matplotlib（见第五层） |

### 4.2 自动验证算法

```
for each state (s1,s2,s3) in 8_states:
    L_before = s1 xor s2 xor s3
    for each switch Si in {S1, S2, S3}:
        flip Si                       # 翻转该位
        L_after  = new_s1 xor new_s2 xor new_s3
        assert L_before != L_after    # 必须翻转
```

验证规模：8 个初始状态 × 3 个开关 = 24 次测试。预期结果：全部 PASS。

## 六、结果分析层（评分：结果解释 2 分）

报告里要讲清三点（对应评分点"设计逻辑、验证过程、结果正确性"）：

1. **设计逻辑**：L = S1⊕S2⊕S3 如何由状态空间和真值表导出——不是猜的，是逐行按"相邻行 L 必变"递推出来的。
2. **表达式最简性**：异或形式 L = S1⊕S2⊕S3 比展开的与或式（4 项）简洁得多，且天然满足任意位翻转 → 整体翻转的性质。对于三控开关问题，异或是数学上最自然的表述。
3. **验证结果**：24 种翻转 100% 通过，程序输出与手工真值表完全一致。

**加分项 · 偶控开关反例**：若将开关数改为偶数（如二控开关），不存在满足"任一开关变化 → 输出变化"的布尔函数——因为偶数次翻转后应回到原状态，但两次不同的单次翻转会经过不同的中间状态。这反过来说明三控开关之所以成立，恰恰因为 3 是奇数。
