# 七、总结与反思

通过本次任务二的实践，主要收获如下：

（1）**数学建模能力**：将一个实际的工程需求（三控开关）转化为离散数学问题，通过枚举状态空间、建立真值表、推导布尔表达式完成逻辑建模。体会到"异或"运算不是凭空选择的——它是由"任一开关变化 → 输出变化"这一设计约束自然导出的唯一解。

（2）**编程实现能力**：使用 Python 从零实现了真值表自动生成、交互模式、自动验证和可视化四个模块。过程中遇到了 Windows 控制台编码不兼容 emoji 的问题，通过替换为 ASCII 符号解决；也遇到了 matplotlib 中文不显示的经典问题，通过配置系统字体和清除缓存解决。

（3）**可视化表达能力**：学会了用 matplotlib 绘制三维散点图、热力图等，理解了三类图（结构 / 过程 / 结果）各自的定位——真值表揭示静态映射，状态空间图揭示拓扑结构，切换路径图揭示动态行为，验证矩阵给出量化结论。四张图相互配合，形成完整的可视化论证链条。

不足与改进方向：
- 当前程序为命令行脚本，若改为简单的图形界面（如 tkinter）交互体验会更好。
- 三控开关逻辑可以进一步用硬件描述语言（如 Verilog）实现，对比软件实现更有工程深度。
- 后续统稿阶段需注意统一三位同学的图表风格，确保报告整体一致性。
